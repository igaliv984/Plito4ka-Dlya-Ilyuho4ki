# Kerama Luxe

Статический каталог и лёгкий интернет-магазин плитки без корзины, без онлайн-оплаты и без тяжёлого backend. Проект собран под `Astro + TypeScript + Tailwind CSS`, ориентирован на быстрый рендер, хорошую индексацию и бесплатный хостинг на Netlify.

## Архитектура

- `Astro` генерирует все страницы статически: главную, каталог, категории, карточки товаров, FAQ и служебные страницы.
- Данные загружаются на этапе сборки, а не в браузере пользователя. Основной источник: опубликованная `Google Sheets`.
- Если Google Sheets ещё не подключена, сайт автоматически использует демо-данные из `src/data/sample`.
- Лиды собираются через `Netlify Forms`, поэтому отдельный backend не нужен.
- Калькулятор плитки работает целиком на клиенте, без API и серверных функций.
- SEO-файлы `robots.txt` и `sitemap.xml` генерируются автоматически.

Почему это подходит для Netlify Free:

- нет постоянного backend и дорогих серверных вызовов;
- сборка простая и дешёвая;
- страницы заранее готовы к индексации;
- сайт быстро открывается даже на мобильных устройствах.

- `/delivery-payment/` — доставка и оплата

## Google Sheets как мини-CMS

Рекомендуемая схема подробно описана в [docs/google-sheets-schema.md](/C:/Users/aliv5/Documents/Codex/2026-04-24/senior-full-stack-ui-ux-seo/docs/google-sheets-schema.md).

Коротко:

- `products` — карточки товаров
- `categories` — категории каталога
- `creatives` — дополнительные изображения и баннеры
- `pages` — тексты статических страниц
- `faq` — вопросы и ответы
- `seo` — SEO-override для ключевых страниц
- `settings` — телефон, адрес, валюта, карта, часы работы

## Как подключить Google Sheets

1. Создайте Google Таблицу по схеме из `docs/google-sheets-schema.md`.
2. Для каждого листа опубликуйте таблицу или используйте публичный CSV-экспорт.
3. Скопируйте ID таблицы из URL.
4. Для каждого листа получите `gid`.
5. Заполните `.env` на основе `.env.example`.

Если `.env` не заполнен, сайт продолжит работать на локальных демо-данных.

### Режимы источника товаров

Поддерживаются два режима:

- `GOOGLE_SHEETS_PRODUCTS_FORMAT=catalog`
- `GOOGLE_SHEETS_PRODUCTS_FORMAT=stock_report`

`catalog` подходит для полноценной mini-CMS со своими полями цены, фото, SEO и текстов.

`stock_report` подходит под выгрузку остатков, похожую на файл `Остатки_Испания,_Италия,_Иран,_Индия_от_17_04_2026.xlsx`. В этом режиме сайт автоматически разбирает страну, бренд, коллекцию, название, код, артикул и остатки по складам.

Самый простой путь:

1. Положите ваш файл в `data/catalog.xlsx`.
2. Поставьте `GOOGLE_SHEETS_PRODUCTS_FORMAT=stock_report`.
3. Если нужно, добавьте `LOCAL_XLSX_PATH=data/catalog.xlsx`.

В этом режиме Google Таблица не нужна вообще. Если файла `data/catalog.xlsx` нет, проект может по-прежнему работать через Google Sheets или демо-данные.

Важно: в выгрузке остатков обычно нет качественных фото, цен и продающих описаний. Поэтому для красивого и конверсионного каталога такие данные лучше дополнять отдельно.

## Деплой на Netlify

1. Загрузите проект в GitHub, GitLab или Bitbucket.
2. В Netlify создайте новый сайт из репозитория.
3. Укажите:
   - Build command: `npm run build`
   - Publish directory: `dist`
4. В разделе Environment variables добавьте:
   - `PUBLIC_SITE_URL`
   - `GOOGLE_SHEETS_ID`
   - `GOOGLE_SHEETS_PRODUCTS_FORMAT`
   - `GOOGLE_SHEETS_*_GID` для всех листов
5. Запустите первый deploy.
6. Проверьте, что форма заявок включена в разделе Forms.

## Как владельцу обновлять контент

### Товары и цены

1. Откройте лист `products`.
2. Измените цену, наличие, описание или CTA.
3. Сохраните таблицу.
4. Запустите новый deploy в Netlify.

### Тексты страниц

1. Откройте лист `pages`.
2. Обновите поля `title`, `content`, `seo_title`, `seo_description`.
3. Запустите новый deploy.

### FAQ

1. Откройте лист `faq`.
2. Добавьте или измените строки.
3. Запустите новый deploy.

### Изображения

Самый простой рабочий вариант:

1. Загружать изображения в Cloudinary, Supabase Storage или другой публичный хостинг файлов.
2. Копировать готовую прямую ссылку на изображение.
3. Вставлять ссылку в `main_image`, `gallery_images` или лист `creatives`.

Для `gallery_images` удобно хранить несколько URL в одной ячейке, каждый с новой строки.

### Карта на странице контактов

Для карты используются два поля:

- `map_link` для кнопки открытия карты;
- `map_embed_url` для iframe с картой на странице контактов.

Если `map_embed_url` не указан, сайт попробует собрать встраиваемую ссылку Яндекс Карт из адреса автоматически.

## SEO-реализация

- мета-теги `title`, `description`, canonical;
- Open Graph и Twitter Cards;
- Schema.org `Product` на карточках товара;
- Schema.org `BreadcrumbList` на товарах и категориях;
- Schema.org `FAQPage` на FAQ-странице;
- чистые URL;
- `robots.txt`;
- `sitemap.xml`.
